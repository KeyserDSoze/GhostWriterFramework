# Writing flow

> From script to final paragraph, with ghostwriters and editing tools.

## The three stages: Script → Draft → Final

Each paragraph can go through three stages:

1. **Script** — the scene's blueprint. A visual editor with nestable blocks: sections, dialogues (with lines, actions, emotions), secrets, and references to characters, locations, items, factions, events. Curly braces `{ }` are containers, square brackets `[ ]` are elements. You don't write the syntax by hand: you build it with buttons.
2. **Draft** — the first prose, generated from the script with AI or written by hand.
3. **Final** — the definitive paragraph, polished from the draft.

They are all connected: from the script you generate/regenerate the draft; from the draft you generate the final; and you can also build the script starting from the draft or the existing paragraph. The chapter's "Scripts" and "Drafts" pages show every paragraph with its state.

## Ghostwriters

A **ghostwriter** is a writer with their own style (tone, voice, rhythm, vocabulary, influences, what to avoid, etc.). You create them in the book's Ghostwriter section and they are saved in the repository.

- If you don't pick a ghostwriter, the book's **default writing style** is used (`guidelines/writing-style.md`).
- You can pick a ghostwriter when you generate script/draft/final; it ends up in the file's metadata.
- The style can also have a **per-chapter** exception.

## Improve and Synonym

While writing prose (paragraph, draft, canon sheet):

- **Improve**: select a part (or nothing for the whole text) → a preview opens with an old/new diff. You can **Regenerate** or **Apply**. It always gets context: book summary, chapter summary (if any), the text, and the style/ghostwriter.
- **Synonym**: select 1-3 words → it suggests **3 alternatives**. Pick one or ask for **3 more** (excluding the ones already seen). It preserves the spaces around the word.

## The text menu (right-click / selection)

The menu is available **everywhere on the site**:

- **Desktop**: right-click. On an editable field you get Select all, Copy, Cut, Paste, Special paste (plus Improve/Synonym if it's a prose editor). On non-editable selected text you get Copy and Special paste.
- **Mobile**: select text normally; a small wand button appears near the selection; tap it to open the menu. (No long-press, so the native selection stays intact.)

**Special paste**: keeps a history of the last 20 copies/cuts made in the site (also saved to Drive), so you can re-paste something you copied earlier.

## Copilot and voice

- The **Copilot** is the assistant: it knows the page context (book/chapter/paragraph) and can create, edit, summarize, search, handle branches, etc. It replies in the app's language.
- The **ghost** opens **live voice** mode: you talk and it replies by voice. You can also say "read me chapter 1" and it reads the text.
- Text-to-speech uses the voice set in Settings; when reading the book it reads only the body (not the metadata), unless you ask for the metadata.
